## 🚨 When Alerts Are Sent

The system sends Telegram alerts in **4 scenarios**:

### 1. Overall Status Change

```python
# Example: Position turns from BULLISH to BEARISH
Previous: BULLISH
Current:  BEARISH
→ Alert: "Status changed: BULLISH → BEARISH"
```

**Anti-spam logic:** Only alerts on **changes**, not every check.

### 2. Important Indicators Change (MA10, OTT) ⭐

```python
# Example: MA10 changes from BULLISH to BEARISH
Previous Important: "BULLISH,BULLISH" (MA10, OTT)
Current Important:  "BEARISH,BULLISH"
→ Alert: "Important Indicators Changed: MA10: BULLISH → BEARISH"
```

**Important Indicators:**
- **MA10** (EMA 10) - Fast moving average, early trend indicator
- **OTT** - Optimized Trend Tracker, trend direction

**Why these indicators?**
- MA10: First to react to price changes (early warning)
- OTT: Reliable trend confirmation (filters noise)
- Together: Early warning + Confirmation = Better timing

**Example Scenarios:**

**MA10 Early Warning:**
```
Position: ETHUSD SHORT
Previous: MA10=BEARISH, OTT=BEARISH
Current:  MA10=BULLISH, OTT=BEARISH
→ Alert: "Important Indicators Changed: MA10: BEARISH → BULLISH"
💡 Meaning: Early warning - MA10 detected upward momentum
```

**OTT Trend Confirmation:**
```
Position: BTCUSD LONG
Previous: MA10=BULLISH, OTT=BULLISH
Current:  MA10=BULLISH, OTT=BEARISH
→ Alert: "Important Indicators Changed: OTT: BULLISH → BEARISH"
💡 Meaning: Trend confirmation changed - OTT detected downtrend
```

**Both Change (Strong Signal):**
```
Position: SOLUSD SHORT
Previous: MA10=BEARISH, OTT=BEARISH
Current:  MA10=BULLISH, OTT=BULLISH
→ Alert: "Important Indicators Changed: MA10: BEARISH → BULLISH, OTT: BEARISH → BULLISH"
💡 Meaning: Complete reversal - both early warning and trend confirmation
```

### 3. Stop Loss Warning

```python
# Example: Position down 5% or more
PnL: -5.2%
Threshold: -5.0%
→ Alert: "Stop Loss Warning: PnL -5.2% (threshold: -5.0%)"
```

**Configurable threshold:** Default is `-5%`

### 4. Take Profit Warning

```python
# Example: Position up 10% or more
PnL: +12.5%
Threshold: +10.0%
→ Alert: "Take Profit Warning: PnL +12.5% (threshold: +10.0%)"
```

**Configurable threshold:** Default is `+10%`

## 🧠 Alert Logic Flow

```
┌─────────────────────────────────────────────────────┐
│  Scheduler runs every 1 hour (default)              │
│  (Configurable: 1-24 hours via .env)                │
└─────────────────┬───────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────┐
│  PositionMonitor.check_all_positions()              │
│  - Fetches all OPEN positions from database         │
└─────────────────┬───────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────┐
│  For each position:                                 │
│  1. Fetch live market data (CCXT/yfinance)          │
│  2. Calculate technical signals (6 indicators)      │
│  3. Determine overall status (BULLISH/BEARISH)      │
│  4. Calculate PnL %                                 │
│  5. Check Important Indicators (MA10, OTT)          │
└─────────────────┬───────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────┐
│  _should_send_alert() - Check conditions:           │
│  □ Overall status changed from last check?          │
│  □ Important indicators (MA10/OTT) changed?         │
│  □ PnL < -5% (stop loss)?                           │
│  □ PnL > +10% (take profit)?                        │
└─────────────────┬───────────────────────────────────┘
                  │
        ┌─────────┴─────────┐
        │                   │
        ▼                   ▼
     YES (alert)         NO (skip)
        │                   │
        ▼                   ▼
┌─────────────────┐  ┌──────────────────┐
│ Format message  │  │ Log: "No alert   │
│ Send via        │  │ needed"          │
│ Telegram        │  └──────────────────┘
└────────┬────────┘
         │
         ▼
┌─────────────────────────────────────────────────────┐
│  Update database:                                   │
│  - last_signal_status = current_status              │
│  - last_important_indicators = MA10,OTT             │
│  - last_checked_at = now                            │
└─────────────────────────────────────────────────────┘
```

**Refresh Frequency:** Every 1 hour (default, configurable)  
**Timezone:** UTC  
**Next Check:** Shown in dashboard (Settings → Scheduler Status)

## 🔍 Key Functions

### 1. `_should_send_alert()` (src/monitor.py)

**Location:** Line ~236

```python
def _should_send_alert(
    self,
    position: Position,
    current_status: str,
    pnl_pct: float,
    signal_states: dict,
) -> tuple[bool, str]:
    """Determine if alert should be sent."""
    
    previous_status = position.last_signal_status
    previous_important = position.last_important_indicators
    
    # Get current important indicators
    current_ma10 = signal_states.get("MA10", "NEUTRAL")
    current_ott = signal_states.get("OTT", "NEUTRAL")
    current_important = f"{current_ma10},{current_ott}"
    
    # Check 1: Overall status change
    if current_status != previous_status:
        return True, f"Status changed: {previous_status} → {current_status}"
    
    # Check 2: Important indicators change (MA10 and/or OTT)
    if current_important != previous_important:
        changes = []
        prev_parts = previous_important.split(",")
        curr_parts = current_important.split(",")
        
        if prev_parts[0] != curr_parts[0]:
            changes.append(f"MA10: {prev_parts[0]} → {curr_parts[0]}")
        if prev_parts[1] != curr_parts[1]:
            changes.append(f"OTT: {prev_parts[1]} → {curr_parts[1]}")
        
        if changes:
            return True, f"Important Indicators Changed: {', '.join(changes)}"
    
    # Check 3: Stop loss
    if pnl_pct < -5.0:
        return True, f"Stop Loss Warning: PnL {pnl_pct:.1f}%"
    
    # Check 4: Take profit
    if pnl_pct > 10.0:
        return True, f"Take Profit Warning: PnL {pnl_pct:.1f}%"
    
    return False, "No significant change"
```

### 2. `_format_alert_message()` (src/monitor.py)

**Location:** Line ~360

```python
def _format_alert_message(
    self,
    position: Position,
    signal_states: dict,
    current_price: float,
    pnl_pct: float,
    reason: str,
) -> str:
    """Format Telegram message with signals and PnL."""
    
    # Build message with emojis and formatting
    message = f"🚨 {position.pair} - {position.position_type} Alert\n\n"
    message += f"Timeframe: {position.timeframe}\n\n"
    message += f"💰 Price:\n"
    message += f"Entry: ${position.entry_price:,.2f}\n"
    message += f"Current: ${current_price:,.2f}\n"
    
    # Add PnL with emoji
    pnl_emoji = "🟢" if pnl_pct >= 0 else "🔴"
    message += f"{pnl_emoji} PnL: {pnl_pct:+.2f}%\n\n"
    
    # Add signal breakdown
    message += "📈 Signals:\n"
    for indicator, status in signal_states.items():
        if indicator not in ['values']:
            emoji = "✅" if status in ["BULLISH", "OVERBOUGHT"] else "❌" if status in ["BEARISH", "OVERSOLD"] else "➖"
            message += f"{emoji} {indicator}: {status}\n"
    
    # Add reason
    message += f"\n⚠️ Reason: {reason}\n"
    
    return message
```

## ⚙️ Customization

### Change Alert Thresholds

**In `src/monitor.py`:**

```python
class PositionMonitor:
    def __init__(
        self,
        telegram_enabled: Optional[bool] = None,
        pnl_warning_threshold: float = -5.0,      # Change this
        pnl_take_profit_threshold: float = 10.0,  # Change this
    ):
        self.pnl_warning_threshold = pnl_warning_threshold
        self.pnl_take_profit_threshold = pnl_take_profit_threshold
```

**Or via environment variables:**

```bash
# .env
PNL_WARNING_THRESHOLD=-3.0    # Alert at -3% instead of -5%
PNL_TAKE_PROFIT_THRESHOLD=15.0 # Alert at +15% instead of +10%
```

### Change Check Interval

**In `.env` (Recommended):**

```bash
# .env
MONITOR_INTERVAL=3600   # Check every 1 hour (default)
MONITOR_INTERVAL=7200   # Check every 2 hours
MONITOR_INTERVAL=14400  # Check every 4 hours
```

**Why 1 hour default?**
- Matches h1 timeframe analysis
- Faster detection of Important Indicators changes
- Better for active position monitoring
- Still reasonable API call frequency

**Trade-offs:**

| Interval | Pros | Cons | Best For |
|----------|------|------|----------|
| **1 hour** (Default) | ⚡ Fast alerts<br>📊 Current data | 🔋 More API calls | Active monitoring, h1 timeframe |
| 2-4 hours | ⚖️ Balanced | ⏰ Moderate delay | Day trading, h4 timeframe |
| 6-12 hours | 🔋 Fewer API calls | 🐌 Slower reaction | Swing trading, d1 timeframe |
| 24 hours | 🔋 Minimal API calls | 🐌 Daily only | Long-term positions |

**Note:** The scheduler runs independently. Changes take effect on next restart.

### Disable Specific Alert Types

**Modify `_should_send_alert()` in `src/monitor.py`:**

### Too Many Alerts

**Solution 1:** Increase check interval

```bash
# .env
CHECK_INTERVAL_HOURS=6  # Check every 6 hours instead of 4
```

**Solution 2:** Widen PnL thresholds

```bash
# .env
PNL_WARNING_THRESHOLD=-10.0   # Alert at -10% instead of -5%
PNL_TAKE_PROFIT_THRESHOLD=20.0 # Alert at +20% instead of +10%
```

**Solution 3:** Only alert on status changes

Edit `src/monitor.py` and comment out PnL checks (see customization section).

**Check:** Database `last_signal_status` field

```sql
-- Check if status is being tracked

FROM positions
WHERE status = 'OPEN';
```

If `last_signal_status` is NULL, the field may not be initialized. Run migration:

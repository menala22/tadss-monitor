### Task 4: API Security - Add Authentication

**Priority:** 🔴 **CRITICAL**
**Effort:** 30 minutes
**Status:** ⏳ Pending

#### Problem
The API (port 8000) is currently open to the internet with **no authentication**. Anyone who knows your VM IP can:
- View all your trading positions
- Add fake positions
- Close or delete your positions
- Access sensitive trading data

#### Solution

**Add API Key Authentication:**

1. **Add to `.env`:**
   ```bash
   API_SECRET_KEY=your_random_secret_here
   ```

2. **Add middleware to `src/main.py`:**
   ```python
   from fastapi import Header, HTTPException

   def verify_api_key(x_api_key: str = Header(None)):
       if x_api_key != settings.api_secret_key:
           raise HTTPException(status_code=401, detail="Invalid API Key")
   ```

3. **Protect sensitive endpoints:**
   - `/api/v1/positions/**` - Require API key
   - `/api/v1/positions/scheduler/**` - Require API key
   - `/health` - Keep public (for monitoring)

4. **Update dashboard to send key:**
   ```python
   headers = {"X-API-Key": os.getenv("API_SECRET_KEY")}
   requests.get(f"{API_BASE_URL}/positions/open", headers=headers)
   ```

5. **Update production script:**
   ```bash
   export API_SECRET_KEY
   ```

#### Acceptance Criteria
- [ ] API key generated and stored in `.env`
- [ ] Middleware added to FastAPI
- [ ] Sensitive endpoints protected
- [ ] Dashboard sends API key
- [ ] Test: API calls without key return 401
- [ ] Test: API calls with key succeed
- [ ] Document how to use API key

#### Security Notes
- API key should be at least 32 characters
- Use `openssl rand -hex 32` to generate
- Never commit `.env` to git
- Rotate key periodically

### Task 5: Firewall Restriction (Optional Hardening)

**Priority:** Medium
**Effort:** 15 minutes
**Status:** ⏳ Pending

#### Problem
Port 8000 is open to `0.0.0.0/0` (entire internet). While API authentication helps, restricting access adds defense in depth.

#### Solution

**Restrict Google Cloud Firewall to Your IP:**

1. **Find your home/office IP:**
   ```bash
   curl https://ifconfig.me
   # or visit: https://whatismyipaddress.com/
   ```

2. **Google Cloud Console → VPC Network → Firewall:**

3. **Edit `allow-tadss-api` rule:**
   - Change source from `0.0.0.0/0` to `YOUR_IP/32`
   - Save

4. **Update when IP changes:**
   - Most residential IPs are dynamic
   - Update rule when IP changes

#### Trade-offs

| Option | Pros | Cons |
|--------|------|------|
| **Open to internet** | No IP updates needed | Less secure |
| **Restricted to IP** | More secure | Must update when IP changes |
| **Both (auth + firewall)** | Best security | More complex |

#### Acceptance Criteria
- [ ] Your current IP added to firewall rule
- [ ] API still accessible from your location
- [ ] Document how to update IP
- [ ] Consider IP update automation
